## The theory

A time series is modeled as a sum of **harmonic pairs**. One pair is two sinusoids at two periods
`T1` and `T2`:

```
f(t) = a1*cos(2*pi*t/T1) + b1*sin(2*pi*t/T1) + a2*cos(2*pi*t/T2) + b2*sin(2*pi*t/T2)
```

Given the two periods, the four coefficients are found by ordinary least squares. The periods
themselves are unknown, so a global search (differential evolution) finds the pair that explains the
most variance of whatever signal is left. Then that pair is subtracted and the search runs again on
the residual. Pairs are added one at a time until the Bayesian Information Criterion stops improving.

Three ideas make this more than a plain Fourier fit.

**Why pairs.** Two sinusoids at close periods sum to one oscillation whose amplitude rises and falls
(a beat). So a pair can represent either two separate cycles or one cycle with a changing amplitude.
That is something a fixed-frequency Fourier basis cannot express.

**Classification.** Each component is labeled by its period `T` relative to the series length `n`:

```
T > n            -> trend       (slower than the whole record)
T < (2/3) n      -> cycle       (at least about 1.5 periods fit inside the record)
(2/3) n < T <= n -> grey zone   (ambiguous: a weak trend or a very long cycle)
```

"Noise" is reserved for the small residual left after decomposition, not for a component.

**Model selection by BIC.** Adding pairs always lowers the fit error, so something has to decide when
to stop. The criterion is

```
BIC = 4j * ln(N) + N * ln(2*pi*e*sigma^2)
```

for `j` pairs on `N` points with residual variance `sigma^2`. The first term penalizes complexity, the
second rewards fit. The decomposition halts when a new pair raises BIC.

The fit is the sum of all the pairs:

![fit](images/02_fit.png)

and each pair is a component in its own right:

![components](images/03_components.png)

For the full derivations, the Bretthorst Bayesian periodogram, and the parameter-count caveats, read
[knowledge/01-methodology.md](knowledge/01-methodology.md) through
[knowledge/03-amplitude-modulation.md](knowledge/03-amplitude-modulation.md).

## Input data: the Series

The model works on one series at a time, two columns: `ds` (an integer step or a regular datetime) and
`y` (the value). Load it from anything.

```python
Series.from_json("data/before_1.json")            # columnar {"ds": [...], "y": [...]}
Series.from_csv("series.csv")                      # ds,y header
Series.from_excel("book.xlsx", sheet=0)            # ds and y columns
Series.from_dataframe(df, ds_col="ds", y_col="y")  # a pandas DataFrame
Series.from_array(y)                               # a bare array; ds defaults to 0..N-1
```

`series.validate()` checks the contract (regular spacing, no duplicate or non-finite values) and raises
a precise error if something is wrong. `series.concat(other)` joins two series end to end, which is how
the bundled before runs combine into one composite.

## Fitting a model

```python
model = HarmonicModel(min_r2_gain=0.02, solver="lstsq").fit(series)
print(model.n_pairs_, model.r2_)
```

`HarmonicModel` is sklearn-style: configure it, call `fit`, then read attributes and call methods. The
defaults reproduce the original notebook exactly. The options:

| argument | default | what it does |
|----------|---------|--------------|
| `max_pairs` | `None` | cap the number of pairs (None = run until BIC stops) |
| `min_r2_gain` | `0.0` | also stop once a pair adds less than this R2 (anti-overfit guard) |
| `max_period_factor` | `10.0` | period search bounds are `[2, factor * N]` |
| `popsize`, `maxiter` | `25`, `500` | differential-evolution budget (lower = faster, rougher) |
| `solver` | `"normal"` | `"normal"` is the notebook solve; `"lstsq"` is more stable |
| `random_state` | `42069` | seed for the period search |
| `verbose` | `False` | print each pair as it is found |

After `fit` the model exposes `n_pairs_`, `pairs_` (the per-pair results), `fitted_`, `residuals_`,
`r2_`, and `metrics_`.

## How many pairs

This is the one choice that matters most. Each pair raises the cumulative R2, with diminishing returns:

![pair selection](images/07_pair_selection.png)

The faithful default (BIC) keeps going well past the point of useful structure and overfits on clean
signals, which is a documented finding in [knowledge/07-qa-audit.md](knowledge/07-qa-audit.md). The
recommended control is the **R2 gain**: stop once a new pair adds less than a set fraction of variance.

```python
HarmonicModel(min_r2_gain=0.02)      # recommended: stop when a pair adds < 2% R2 (~7 pairs here)
HarmonicModel(min_r2_gain=0.01)      # keep more structure (~12 pairs here)
HarmonicModel(max_pairs=8)           # or a hard cap, if you want an exact count
HarmonicModel()                      # faithful BIC stop (can overfit; matches the paper)
```

Read the right threshold off the elbow of the curve above: pick where the gains flatten.

## Reading the results

`summary()` prints a metrics header and the full per-pair table:

```python
model.summary(grey_zone_label="grey zone")
```

```
==================================================================
HarmonicModel summary  -  series
==================================================================
observations        18260      pairs               7
R2                 0.7851      RMSE           ...
MAE                ...         BIC            ...
------------------------------------------------------------------
components:  trend 1   cycle 6   grey zone 0   (var-amp pairs ...)
==================================================================
```

The pieces:

- `model.analyze()` returns the per-pair DataFrame (periods, amplitudes `A=sqrt(a^2+b^2)`, phases
  `phi=atan2(a,b)`, the three variable-amplitude conditions, and the trend/cycle flags).
- `model.count_components()` returns the trend/cycle/grey-zone counts.
- `model.metrics_` holds R2, RMSE, MAE, MAPE, residual std, BIC, and AIC.

## Plots

Every figure is a one-call method and returns a matplotlib axes (needs the `[viz]` extra). Legends sit
outside the plot, and the components legend is dropped past 20 pairs.

```python
model.plot_series()                 # the raw input
model.plot_fit()                    # actual vs fitted vs residual
model.plot_components()             # all pairs (legend dropped past 20)
model.plot_components(pairs=[1, 3])  # only the pairs you choose (1-based)
model.plot_pair(2)                  # any single pair; separate=True splits its two harmonics
model.plot_residuals()              # residuals only
model.plot_forecast(h, level)       # forecast with intervals
model.plot_variable_amplitude()     # the most beat-like pair and its envelope
model.plot_pair_selection()         # cumulative R2 vs pairs
model.plot_evolution()              # the fit building up as pairs are added
```

The evolution plot shows the reconstruction tightening from one pair to all of them:

![evolution](images/08_evolution.png)

`model.pair_signal(i)` returns a single pair's contribution and `model.cumulative(k)` the fit using
the first `k` pairs, so you can build any custom view. All plot methods accept an optional `window=`
(an int or a `(start, stop)` slice) to zoom and an `ax=` to draw onto an existing axes. The generic
form is `model.plot(kind="fit", ...)`.

## Forecasting

`predict` continues every fitted harmonic past the end of the training window.

```python
fc = model.predict(100, level=[80, 95])
fc["ds"], fc["yhat"], fc["lo_95"], fc["hi_95"]
```

`level` is optional; when given it adds residual-based prediction intervals at each percentage. Because
the harmonic mean is deterministic, the interval width comes from the residual spread, not from
compounding uncertainty.

![forecast](images/04_forecast.png)

The intervals here are a simple residual band. Stronger options (delta method, bootstrap, posterior
predictive) are written up in [knowledge/09-gap-research.md](knowledge/09-gap-research.md).

## Variable amplitude (beats)

When a pair has two close periods it represents one oscillation whose amplitude is modulated by a slow
envelope. The carrier and envelope periods are

```
carrier  T   = 2*T1*T2/(T1 + T2)
envelope T_A = 2*T1*T2/|T1 - T2|
```

`model.plot_variable_amplitude()` finds the most beat-like pair and draws it with its envelope:

![variable amplitude](images/05_variable_amplitude.png)

`model.variable_amplitude_pairs()` returns, for every pair flagged variable-amplitude, the envelope and
carrier components (`envelope_component`, `residual_component`) and their periods. The flag uses three
conditions on `T`, `T_A`, and the series length; see
[knowledge/03-amplitude-modulation.md](knowledge/03-amplitude-modulation.md).

## Comparing two series

The method was first published for a before/after comparison. Fit each series and compare. The visual
overlay and the component counts are available now.

```python
mb = HarmonicModel().fit(before)
ma = HarmonicModel().fit(after)
mb.compare_to(ma, labels=["before", "after"])
mb.count_components(); ma.count_components()
```

![before vs after](images/06_before_after.png)

You read the difference off the structure: how the trend, cycle, and grey-zone counts change, whether
variable-amplitude pairs appear, and how the total R2 and residual variance move. A single-call
`BeforeAfterComparison` with a formal difference test is on the roadmap
([knowledge/05-before-after-comparison.md](knowledge/05-before-after-comparison.md)).

## API reference

### `Series`
`from_json` / `from_csv` / `from_excel` / `from_dataframe` / `from_array`, then `.validate()`,
`.concat(other)`, `.plot(window, ax)`. Attributes: `ds`, `y`, `name`.

### `HarmonicModel`
Construct with the options table above. Methods:

| method | returns |
|--------|---------|
| `fit(y)` | the fitted model (accepts a `Series` or an array) |
| `predict(h, level=None, target_r2=None)` | dict with `ds`, `yhat`, optional `lo_L`/`hi_L` |
| `summary(returns=False, grey_zone_label="noise")` | prints the report, optionally returns the table |
| `analyze()` | per-pair DataFrame |
| `count_components(grey_zone_label="noise")` | trend / cycle / grey-zone counts |
| `variable_amplitude_pairs()` | envelope/carrier split of the beat pairs |
| `plot_series/fit/components/residuals/variable_amplitude/pair_selection(...)` | a matplotlib axes |
| `plot_forecast(h, level=None, ax=None)` | a matplotlib axes |
| `compare_to(other, labels=None, window=4000, ax=None)` | a matplotlib axes |

Attributes after fit: `n_pairs_`, `pairs_`, `fitted_`, `residuals_`, `r2_`, `metrics_`, `series_`.

### `harmonix.core` (the faithful notebook functions)
`estimate_harmonic_pair`, `optimize_harmonic_pair`, `decompose_harmonics_until_bic`,
`forecast_from_harmonic_coeffs`, `analyze_results`, `decompose_variable_amplitude_pair`,
`count_trend_cycle_noise`, plus helpers `classify_single`, `carrier_period`, `envelope_period`.

### `harmonix.metrics`
`r2`, `rmse`, `mae`, `mape`, `bic`, `aic`, `summary_metrics`.

### `harmonix.plot_comparison(models, labels=None, window=4000, ax=None)`
Overlay the fitted models of several series.
